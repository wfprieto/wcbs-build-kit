# Versioning

The Super Build Kit uses semantic versioning.

## Patch

Use patch versions for:

- typo fixes;
- small documentation clarifications;
- doctor false-positive fixes;
- non-breaking adapter wording.

## Minor

Use minor versions for:

- new skills;
- new templates;
- new runtime adapters;
- new doctor checks;
- new system tests;
- new release or install docs.

## Major

Use major versions for:

- source-of-truth order changes;
- APIVR lifecycle changes;
- evidence vocabulary changes;
- release-gate contract changes;
- breaking adapter activation changes.

The repository slug remains `wcbs-build-kit`.
