# Common Workflows

## Audit A Repository

Load:

- `00_start_here/START_HERE.md`
- `00_start_here/SOURCE_OF_TRUTH.md`
- `00_start_here/LOAD_ORDER.md`
- `50_audits/AUDIT_TIER_ROUTER.md`
- `50_audits/CANONICAL_AUDIT_PROTOCOLS.md`

Output: findings, evidence state, release-gate status, remediation plan.

## Plan A Feature

Load:

- `skills/writing-plans/SKILL.md`
- `skills/test-driven-development/SKILL.md` when code is involved
- domain skills from `LOAD_ORDER.md`

Output: zero-placeholder implementation plan with tests and rollback.

## Debug A Failure

Load:

- `skills/diagnosing-bugs-and-feedback-loops/SKILL.md`
- `40_knowledge/SYSTEMATIC_WORKFLOWS.md`

Output: reproduction, hypothesis checks, smallest fix, verification.

## Verify A Release

Load:

- `10_governance/RELEASE_GATES.md`
- `skills/release-readiness-and-ship-gates/SKILL.md`
- `60_templates/RELEASE_READINESS_DASHBOARD_TEMPLATE.md`

Output: release gate table and final APIVR verdict.

## Run A Forensic Audit

Load:

- `skills/cybersecurity-risk-routing/SKILL.md` when security is involved
- `skills/20-pass-protocol/SKILL.md` for high-stakes artifacts
- `60_templates/EVIDENCE_LEDGER_TEMPLATE.md`

Output: chain of evidence, blockers, owners, and remediation plan.
