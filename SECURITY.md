# Security Policy

## Supported Versions

The current `main` branch and latest tagged release are supported for security reports.

## Reporting A Security Issue

Open a private report if GitHub security advisories are enabled, or contact the repository owner through GitHub.

Do not include secrets, live credentials, private customer data, exploit payloads, or third-party target details in public issues.

## Scope

Security reports may include:

- secret exposure;
- unsafe instructions;
- dependency or workflow risk;
- malicious or confusing tool guidance;
- unsafe dual-use routing;
- release or verification bypasses.

## Handling

Security work follows:

- `skills/cybersecurity-risk-routing/SKILL.md`
- `skills/security-incident-response/SKILL.md`
- `skills/supply-chain-and-build-provenance/SKILL.md`
- `60_templates/SECURITY_AUTHORIZATION_AND_SCOPE_TEMPLATE.md`

Unknown, Not Run, or Blocked security evidence cannot be treated as PASS for release.
