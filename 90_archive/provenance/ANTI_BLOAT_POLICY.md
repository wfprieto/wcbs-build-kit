# Anti-Bloat Policy

The Super Build Kit should beat narrower systems without becoming hard to use.

Rules:

- Add a new skill only when an existing canonical skill cannot own the behavior.
- Prefer upgrading canonical files over adding parallel guidance.
- Keep load order minimal and task-routed.
- Archive provenance separately from active instructions.
- Add executable checks for durable claims.
- Treat duplicate active guidance as a maintenance risk.
