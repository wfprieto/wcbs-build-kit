# Readiness Gap Closure Plan

Objective: move `wcbs-build-kit` from a software-ready governance repository into a runtime-proven software system while preserving APIVR and the 16 Elite Build Goals as permanent guardrails.

## Non-Goals

- Do not rename the repository.
- Do not weaken APIVR, evidence states, release gates, or security authorization gates for ease of installation.
- Do not claim runtime support from file presence alone.
- Do not add broad new skills until the execution layer can prove activation and support levels.

## Phases

| Phase | Upgrade | Acceptance Evidence |
|---|---|---|
| 0 | Preserve side-by-side audit and readiness gap | Provenance record exists; manifest and support docs point to the gap. |
| 1 | Separate documented support from verified support | `runtime_adapters/VERIFIED_SUPPORT_LEVELS.md` exists and is checked by doctor. |
| 2 | Add safe real installer modes | Installer supports dry-run, install, update, uninstall, and doctor with explicit destination. |
| 3 | Add isolated runtime fixtures | Codex and Cursor fixtures prove project-local install behavior. |
| 4 | Add install/update/uninstall integration tests | Tests prove ownership manifest, safe update, and bounded uninstall. |
| 5 | Add behavioral activation smoke tests | Startup scenario proves APIVR routing, evidence states, and unsupported-tool honesty. |
| 6 | Add release artifact pipeline | Zip artifact and SHA256 checksums are generated by script. |
| 7 | Add tag-driven release workflow | GitHub Actions runs release checks and uploads release evidence. |
| 8 | Simplify day-one entry | A short `GET_STARTED.md` gives the minimum safe path without hiding advanced controls. |
| 9 | Re-score readiness | Closure report compares updated readiness against the July 14 audit. |

## Success Targets

- Operational readiness improves from 84.39 to at least 92.
- Equal-weight score improves from 90.03 to at least 94.
- Behavioral validation improves from 74 to at least 90.
- Release hygiene improves from 60 to at least 90.
