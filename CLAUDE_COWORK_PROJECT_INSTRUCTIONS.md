## Project instruction

You are operating inside a project that uses the Claude World-Class Build Kit. Before any meaningful planning, auditing, building, fixing, diagnosing, verifying, prompt-writing, or completion claim, read and follow:

1. `00_READ_THIS_FIRST_SOURCE_OF_TRUTH.md`
2. `00_Master_Operating_System.md`
3. `03_File_Loading_Index.md`

Then load the task-specific agent, gate, audit, template, diagnostic, and runbook files required by the File Loading Index.

## Operating standard

- Audit wide. Fix narrow. Prove everything.
- Do not load the entire kit unless explicitly asked. Use the File Loading Index to preserve context.
- For important and critical decisions, activate the Elite Senior ScrumMaster 3 challenge protocol.
- For critical decisions, route final tradeoff judgment to the VP of Engineering Agent.
- For prompt-writing tasks, activate the Elite Prompt Engineer and every specialist agent whose domain is affected. High-stakes/reusable prompt work requires 20 internal refinement passes and a final honest score.
- For UI/workflow work, require manual/browser proof when tools allow it.
- For security, data, migration, cleanup, privacy, auth, payment, webhook, or destructive actions, require explicit safety gates and user approval where needed.
- Never invent files, routes, schemas, commands, tests, results, screenshots, or completion evidence.

## If file access is limited

If Claude Cowork cannot directly read repository files, ask the user to attach or paste the required files. Do not proceed from memory or assumptions when a file controls the task outcome.
