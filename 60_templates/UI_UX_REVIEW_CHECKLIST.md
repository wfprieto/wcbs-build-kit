# UI/UX Review Checklist

Use during APIVR Phase 4 and Phase 5 for user-facing interface work.

## Design Fit

- [ ] Subject, audience and page job are clear.
- [ ] Design choices fit the product rather than a generic template.
- [ ] One signature element is justified.
- [ ] Decoration that does not serve the brief is removed.
- [ ] Layout supports the user's real task.

## Interface Copy

- [ ] Buttons say what happens.
- [ ] Labels use user-facing language.
- [ ] Empty states guide the next action.
- [ ] Errors explain what happened and how to fix it.
- [ ] Success states confirm the completed action.
- [ ] Terms stay consistent across the flow.

## Accessibility

- [ ] Text contrast is acceptable.
- [ ] Focus states are visible.
- [ ] Keyboard navigation reaches meaningful controls.
- [ ] Touch targets are usable.
- [ ] Color is not the only signal.
- [ ] Motion respects reduced-motion settings.
- [ ] Images and icons have appropriate text alternatives.

## Responsive And Visual QA

- [ ] 375px checked.
- [ ] 768px checked.
- [ ] 1024px checked.
- [ ] 1440px checked.
- [ ] No clipped, hidden or overlapping text.
- [ ] Loading, empty, error and success states are designed.
- [ ] Charts have labels, units, legends and fallback where needed.
- [ ] Rendered screenshot or browser review completed when tooling is available.

- Release gate:
- Remaining risk:
- Final verdict: PASS / CONDITIONAL PASS / PARTIAL / FAIL / BLOCKED
