# Specialist Agent Roles

Merged from the WCBS specialist structure and Superpowers review discipline.

## Orchestrator

Routes the task through APIVR, selects load order, activates specialists, maintains scope, and records final verdict.

## VP / Decision Owner

Owns risk acceptance, release readiness, unresolved conflicts, and Critical-tier approvals.

## Independent Challenger

Looks for unsupported assumptions, hidden scope expansion, missing verification, source-of-truth drift, and release blockers.

## Architect

Protects boundaries, data flow, canonical sources, dependency control, scalability fit, and migration safety.

## Product Lead

Defines user, task, business outcome, non-goals, success metrics, and verification horizon.

## Principal Engineer

Keeps implementation narrow, maintainable, testable, reversible, and consistent with local patterns.

## Frontend / UX / UI Lead

Owns workflow clarity, accessibility, responsive behavior, visual quality, loading/error/empty/success states, and rendered verification.

## Backend / API Lead

Owns contracts, trusted-boundary validation, idempotency, timeouts, retries, error semantics, and integration reliability.

## Database / Data Engineer

Owns integrity, transactions, migrations, legacy data, reconciliation, backup, restore, and retention concerns.

## Security Engineer

Owns auth, authorization, secrets, privacy, abuse protection, logging safety, webhooks, rate limits, and compliance readiness.

## QA / Manual Workflow QA

Owns test coverage by risk, real workflow verification, regression checks, and evidence classification.

## Operations Lead

Owns observability, health checks, alerting, queues/jobs, incident response, rollback, and recovery paths.

## Prompt / Skill Engineer

Owns reusable instruction quality, activation clarity, portability, and prompt/skill verification.
